# arxiv_digest
Retrieve and format submissions from [arXiv.org](https://arxiv.org/) into an html file.

### What it does
The purpose of this script is to simplify the process of keeping up-to-date with submissions from [arXiv.org](https://arxiv.org/).
Specifically, `arxiv_digest` tracks the newly uploaded papers to the archives of interest and formats them into a convenient, easily-read html file, that can be opened in a browser or sent via email.
For every paper of interest the script lists the title (that is also a link to the full pdf), tags, and authors:

### Setup
1. Install requirements: `pip install -r requirements.txt`
2. Copy `settings.ini.example` to `settings.ini` and configure your archives, keywords, and email settings.
3. Create a `.env` file with your SMTP credentials:
   ```
   SMTP_USER=you@gmail.com
   SMTP_PASSWORD=your_app_password
   ```
   For Gmail, use an [App Password](https://myaccount.google.com/apppasswords) (requires 2-Step Verification).

### Running locally
```bash
python arxiv_digest.py
```
Output goes to `html_files/`. If email is enabled in `settings.ini`, the report is also sent via email.

### Deploying to a server
The included `deploy.sh` script deploys to a remote server and sets up a daily cron job.

Add server connection details to your `.env`:
```
SERVER_USER=your_username
SERVER_HOST=your_server_ip
APP_PATH=apps/arxiv_digest
```

Then run:
```bash
./deploy.sh
```

This will clone/pull the repo on the server, copy your config files, set up a Python venv, and install a cron job to run daily at 7am ET. Re-run `./deploy.sh` after any changes.
